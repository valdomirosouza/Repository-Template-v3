# Positioning — {Product / Feature Name}

> Copy into `docs/gtm/.../positioning.md` and fill in. Add the Agent-Disclosure Header if agent-drafted.
> **Reviewer:** Product Owner / Marketing | **Status:** Hypothesis until validated.

Positioning makes the product _understandable and differentiated_ in the buyer's mind. Adapted from
April Dunford's framing. Every competitive claim must be sourced or marked `uncertain — verify`
(CLAUDE.md §3.6).

---

## 1. Competitive alternatives

> What would the customer use if we did not exist? (Often a manual process or a status quo, not just
> a named competitor.)

| Alternative                         | What they'd do | Why it falls short |
| ----------------------------------- | -------------- | ------------------ |
| _Do nothing / manual_               |                |                    |
| _Generic CI/CD + ad-hoc governance_ |                |                    |

## 2. Unique attributes

> Capabilities we have that the alternatives do not. Be concrete and verifiable.

- _e.g. two-tier HITL (Spec-as-PR + runtime gateway) as a first-class, enforced control_
- _e.g. machine-checked traceability (service → spec → ADR → SLO → runbook)_

## 3. Value (what the attributes enable)

| Unique attribute | So what? (value to the buyer)                               |
| ---------------- | ----------------------------------------------------------- |
| _enforced HITL_  | _ship agentic features without losing human accountability_ |

## 4. Target market

- **Who cares most about this value:** _the best-fit segment (link ICP in `gtm-brief.md`)_

## 5. Market category

- **Category we play in:** _e.g. "Agentic SDLC platform" / "AI governance for software delivery"_
- **Why this category:** _frames expectations and the comparison set_

## 6. Positioning statement

> For **{target segment}** who **{need / trigger}**, **{product}** is the **{category}** that
> **{primary differentiated value}**, because **{reason to believe / proof}**.
